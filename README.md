 ## Self-Driving Racing Car Using Machine Learning

This project leverages **Python**, **Pygame**, and **Machine Learning libraries** (Pandas, Scikit-learn) to develop Decision Trees that allow vehicles to navigate a racing track autonomously.
<hr>

### :books: Libraries in Use
* **Pandas & NumPy**: For data manipulation and processing.
* **Scikit-learn**: For training and implementing Machine Learning models.
* **Pygame**: Used for the simulation engine, game logic, and visualization.

<hr>

### 💻 Environment

The project is built using the **Anaconda** distribution and utilizes two main **Jupyter Notebooks**:
* **Main Notebook**: For running the simulation and visualizing the car in action.
* **Training Notebook**: Dedicated to testing, refining, and improving the driving models.
<hr>

### :hammer_and_wrench: Project Inner Workings
* The simulation is built in Pygame, providing a realistic environment where cars can move forward, reverse, and steer.
* **Collision System:** The car interacts with two layers—a visual track image and a hidden masking image. This mask defines the boundaries of the road and handles collision detection.
* **Data Collection**: To train the model, driving data is gathered manually. A player controls the car (WASD), and the car’s sensors (4 to 6 sensors) record the distance to the track boundaries at every tick.
* **Training Process**: Once enough laps are completed, the sensor data is fed into Scikit-learn to generate a predictive model.
* **Anti-Overfitting**: Since there is only one track, a script spawns the car in 5–6 different locations with various orientations. This ensures the model learns to drive based on sensor input rather than just memorizing the track layout.

<hr>

### :bar_chart: Results
While the models successfully produce functional Decision Trees, the current best model completes the track with a success rate of approximately 60%. Interestingly, a manually crafted logic tree currently outperforms the models generated by Scikit-learn, highlighting the complexity of autonomous navigation.

<hr>

### 🚀 Future Works & Limitations
* **Mask Precision**: The track mask's low resolution can lead to precision errors in tight corners. Improving the mask quality is a priority.
* **Track Variety**: Introducing multiple track designs to increase data variability and improve model generalization.
* **Path Following**: Implementing an automated "path-following" algorithm to collect cleaner training data, reducing the dependency on human driving skills.
* **Hyperparameter Tuning**: Further adjusting model parameters to optimize performance.
* **Multi-Agent Simulation**: Testing how multiple autonomous models interact and behave when sharing the track simultaneously.
